## API 명세
| Method | Path | Request Header | Request Body | Response Status Code | Response Body |
| --- | --- | --- | --- | --- | --- |
| GET | /lectures | - | - | 200, 404 | list of lectures / [result:{”Not Found”}] |
| GET | /lectures/profId={profId} | - | - | 200, 404 | list of lectures with profId / [result:{”Not Found”}] |
| GET | /lectures/lecName={lecName} | - | - | 200, 404 | list of lectures with lecName / [result:{”Not Found”}] |
| GET | /lectures/lecId={lecId} | - | - | 200, 404 | list of lectures with lecId / [result:{”Not Found”}] |
| POST | /registration | Content-Type: Application/json, Authorization(Token) | lecId | 201/ 404 | “[Created lecture”/ [result:{”Not Found”}] |
| GET | /registration | Content-Type: Application/json, Authorization(Token) |  | 200, 404 | list of lectures / [result:{”Not Found”}] |
| PUT | /students/{stId} | Content-Type: Application/json, Authorization(Token) |  | 200, 404 | studentInfo / [result:{”Not Found”}] |
| POST | /lectures | ContentType: Application/json, Authorization(Token) | lecId | 201/ 404 | “[Created lecture”/  [result:{”Not Found”}] |
| DELETE | /registration | Content-Type: Application/json, Authorization(Token) | lecId | 200 / 404 | “ok” / [result:{”Not Found”}] |

## 참고
- 개별 항목 조회의 경우 여러 path가 등장 할 수 있기 때문에 기능에 따라 전부 작성해주세요.
    - ex) 친구를 id, name, class에 따라 개별 조회 할 경우
        
    - `/friends/:id` 
    - `/friends?name=name`
    - `/friends?class=class`
